How this is calculated

Short, user-facing summary of the pipeline:

  • Headcount is tracked every month for each legal entity (company).
  • Each company is linked to a country; administrator-maintained national averages supply a typical one-way commute distance (km) and mode split (% car, bus, walking/cycling, mixed).
  • Those percentages allocate headcount across transport modes (integer-split so totals match headcount).
  • Distance rules (round-trip, ~20 commute days per month) follow the same assumptions as the CTS mapping preprocessor; mapping applies emission factors to produce tCO₂e — so you should run Map on {{ employee_commuting_display_category }} after import.
  • Imported rows are tagged Data Source = Employee Commuting and appear under {{ employee_commuting_display_category }} in Data Entry (template key {{ employee_commuting_template_key }}).
